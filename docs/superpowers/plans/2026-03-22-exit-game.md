# Exit Game Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let players exit an active game via the Options dialog, with confirmation, terminating the session for all players and showing a dialog to remaining players identifying who ended the game.

**Architecture:** The backend `game:exit` handler and `session:terminated` socket event are already fully implemented. Work is pure client-side: add exit UI to `OptionsDialog`, wire it through `OptionsButton` → `GameScreen`, add `session:terminated` handling in `useSocket`, and show the terminator's name in `GameTerminatedModal`.

**Tech Stack:** TypeScript, React Native, Expo Router, Socket.IO client, i18next

### Task 2: Add `session:terminated` handling to `useSocket`

**Files:**

- Modify: `packages/ui-shared/src/useSocket.ts`

`useSocket` is in `packages/ui-shared/src/useSocket.ts`. It creates a Socket.IO client inside `useEffect`, stores callbacks in `callbacksRef`, and registers socket listeners inside the same `useEffect` closure. Currently it handles `game:events`, `game:state`, `error`, `player:joined`, `player:left`, `player:reconnected` — but NOT `session:terminated`.

- [ ] **Step 1: Add the callback to the options interface**

In `UseSocketOptions` (currently ends at `onPlayerReconnected`), add:

```typescript
  onSessionTerminated?: (data: { message: string; terminatedBy?: string }) => void;
```

- [ ] **Step 2: Add to the callbacksRef**

In the `callbacksRef` object (both the initial value and the update effect), add `onSessionTerminated`.

Initial value (inside `useRef({...})`):

```typescript
    onSessionTerminated,
```

Update effect array destructure and assignment:

```typescript
    onSessionTerminated,
```

```typescript
    onSessionTerminated,
```

- [ ] **Step 3: Register the `session:terminated` listener**

Inside the `useEffect`, after the `player:reconnected` handler, add:

```typescript
newSocket.on('session:terminated', (data) => {
  newSocket.disconnect();
  callbacksRef.current.onSessionTerminated?.(data);
});
```

`newSocket.disconnect()` prevents auto-reconnection attempts (the session is gone). It's called before the callback so the socket is cleanly shut down regardless of what the consumer does.

- [ ] **Step 4: Create `handleExitGame` callback**

```typescript
const handleExitGame = useCallback(() => {
  socket?.emit('game:exit');
  router.replace('/');
}, [socket, router]);
```

`useGame` doesn't currently expose `socket` directly. Check the return value of `useGame`. If `socket` is not returned, use the `emit` function instead, which is returned as part of the `useSocket` result and already available. Looking at the current `useGame` return: it doesn't expose `socket` or `emit`. The simplest approach is to add an `onExit` action to `useGame` (similar to `onBid`, `onPass`, etc.):

In `useGame.ts`, add alongside the other action callbacks:

```typescript
const onExit = useCallback(() => socket?.emit('game:exit'), [socket]);
```

And return it:

```typescript
    onExit,
```

Then in `GameScreen`, `handleExitGame` becomes:

```typescript
const handleExitGame = useCallback(() => {
  onExit();
  router.replace('/');
}, [onExit, router]);
```
